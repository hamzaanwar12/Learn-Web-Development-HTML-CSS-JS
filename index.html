  2. Introduction

BT is a distributed database which allows users to store, transfer and fetch data without authorization of a centralized authority. It purely works on the basis of global consensus that do not allow any user to change data on the network without consent of all other users.


Blockchain:
It is actually a string of blocks that contain data record entered by the users. The data in blockchains is not stored in the form of tables as in ordinary databases rather it is recorded in the form of some codes with the help of cryptographic encryption algorithms. Once a block is
completely filled it is added to the chain that links it to the previous blocks in a chronological method. All users on the network have the copy of the database that makes this technology transparent and trustworthy.

Moreover, high level of complex encryptions makes it nearly impossible to cyber attack which make it tamper proof and immutable. Potential applications of blockchain technology includes decentralizeddatabases, smart contracts, distributed applications, cloud computing and secure, smart and transparent way of transactions which make the base of cryptocurrencies. The fundamental and initial reason for the development of blockchains was a distributed and secure data base. This idea for the first time was proposed by Stuart Haber and W. Scott Stornetta in 1991 [1] . The white paper comprehension of this technology conveys an idea of cryptographically encrypting blocks of data in the form of chronologically linked chains. This technology appends the idea of timestamping the data record entered in the database. This encrypting of data and blocks brings up the concept of hash values.
